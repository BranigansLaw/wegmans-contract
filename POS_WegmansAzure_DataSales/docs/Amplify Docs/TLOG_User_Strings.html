Appendix B: TLOG User Strings

This section contains the details for all the TLOG User Strings to be used by the TGCS Mobile POS, Consumer and Associate solutions. Note that a unique Originator of X’11’ will be used, thus allowing Subtypes to start a X’01’ without duplicating any previous messages. It is assumed that “99” strings for TLOGS can be up to 508 bytes long.

The property enable.TLog.logging=false is defined in mobileserver.properties. If this property is set to true, the following user 99 strings will be logged as part of the sales transaction within the ACE TLOG.

Note: The Originator and SubType fields listed in all of the user strings above will be ASCII for SA (not packed as described for ACE).

Mobile Transaction Started (SubType X’10’)

This user string is generated to identify a transaction as one that was initiated as a Mobile Shopper transaction through the mobile device.

Field Name Type Length Description
TYPE PD 1 String type = X’99’ for user data.
Originator PI 1 X’11’ for Mobile Exceptions.
Sub Type PI 1 X’10’ for Mobile Transaction started.
Terminal Number ASCII Var (Max 4 chars) Terminal Number.
Customer Loyalty ID ASCII Var (Max 15 chars) The customer Loyalty ID.

Associate Mobile Transaction Started (SubType X’11’)

This user string is generated to identify a transaction as one that was initiated as a Mobile Shopper transaction through the mobile device.

Originator PI 1 X’11’ for Mobile Exceptions.
Sub Type PI 1 X’11’ for Mobile Transaction started.

Add Item Button Pressed (SubType X’01’)

This user string is generated when an item exception is displayed on the POS GUI exception list screen and the operator presses the Add To Transaction button.

Field Name Type Length Description
TYPE PD 1 String type = X’99’ for user data.
Originator PI 1 X’11’ for Mobile Exceptions.
Sub Type PI 1 X’01’ for Add Item.
Operator ID ASCII Var (Max 10 chars) Operator ID.
Terminal Number ASCII Var (Max 4 chars) Terminal Number.
Transaction Number ASCII Var (Max 4 chars) Transaction Number.
Exception Type ASCII Var (Max 60 chars) The type of exception that initiated this action. They could include (but are not limited to):
ITEM_WEIGHT_REQUIRED
ITEM_PRICE_REQUIRED
ACCOUNT_NUMBER_REQUIRED
Exception Code ASCII Var (Max 4 chars) A code to identify the Exception Type. The codes will be as follows for the associated Exception (but could have additional options):
209 = ITEM_WEIGHT_REQUIRED
206 = ITEM_PRICE_REQUIRED
201 = ACCOUNT_NUMBER_REQUIRED
100 = UNDEFINED (Any exception that currently does not have a code associated with it.)
Item Num ASCII Var (Typical Max 30 chars) Item/coupon number if one exists.

Where:

  • ITEM_WEIGHT_REQUIRED - The item is configured to require a scale weight in the ACE item record file.
  • ITEM_PRICE_REQUIRED - The item is configured to require a price in the ACE item record file.
  • ACCOUNT_NUMBER_REQUIRED - The item requires the entry of an account number. An example is gift card activation.

This user string is generated when an item exception is displayed on the POS GUI exception list screen and the operator presses the Void From Transaction button.

Field Name Type Length Description
TYPE PD 1 String type = X’99’ for user data.
Originator PI 1 X’11’ for Mobile Exceptions.
Sub Type PI 1 X’02’ for Void Item.
Operator ID ASCII Var (Max 10 chars) Operator ID.
Terminal Number ASCII Var (Max 4 chars) Terminal Number.
Transaction Number ASCII Var (Max 4 chars) Transaction Number.
Exception Type ASCII Var (Max 60 chars) The type of exception that initiated this action. They could include (but are not limited to):
EAS_ITEM
AGE_RESTRICTED_ITEM
Exception Code ASCII Var (Max 4 chars) A code to identify the Exception Type. The codes will be as follows for the associated Exception:
203 = EAS_ITEM
202 = AGE_RESTRICTED_ITEM
100 = UNDEFINED (Any exception that currently does not have a code associated with it.)
Item Num ASCII Var (Typical Max 30 chars) Item/coupon number if one exists.
Current Scale Weight ASCII Var (Max 6 chars) The current Register Scale Weight if one exists.
Minimum Age Restriction ASCII Var (Max 2 chars) Minimum Age required to purchase this item if one exists.

Where:

  • EAS_ITEM - The item is configured as an “EAS” item in the item record file.
  • AGE_RESTRICTED_ITEM - The item is marked as age restricted within the ACE item record file.

Item Added During Audit (SubType X’07’)

This user string is generated when an item was scanned during an audit that was not already in the transaction (for example, it failed an audit).

Field Name Type Length Description
TYPE PD 1 String type = X’99’ for user data.
Originator PI 1 X’11’ for Mobile Exceptions.
Sub Type PI 1 X’07’ for Item Added during Audit.
Operator ID ASCII Var (Max 10 chars) Operator ID.
Terminal Number ASCII Var (Max 4 chars) Terminal Number.
Transaction Number ASCII Var (Max 4 chars) Transaction Number.
Item Num ASCII Var (Max 14 chars) The Item number added during an audit (for example, the item was not previously entered by the mobile shopper, but was found in their basket).

Operator Ended Audit (SubType X’09’)

This user string is generated when an operator ends an audit without completing it. The audit will be marked as failed.

Field Name Type Length Description
TYPE PD 1 String type = X’99’ for user data.
Originator PI 1 X’11’ for Mobile Exceptions.
Sub Type PI 1 X’09’ for Operator Ended Audit.
Audit state ASCII 1 X’32’ (audit failed)
